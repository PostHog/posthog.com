## Q1 2026 objectives

Data modeling

### Objective 1
- Data modeling as a first-class citizen in app
  - Lineage view 
  - DAG execution + matviews are STABLE with better error surfacing, logging, and alerting

### Objective 2
- Highest priority modeling features shipped
  - Run all downstream models, run all upstream, etc.
  - Model tests
  - Models backed by Git and syncable with GitHub

### Objective 3
- Differentiators against competitors
  - Field level lineage that feels nice
  - Custom PostHog node types (e.g. models as workflow triggers)

Endpoints

### Objective 1
- Endpoints GA
  - Pricing finalized and rolled out

### Objective 2
- Endpoints integrate with managed warehouse
  - Materialized endpoint execution goes through managed warehouse compute

### Objective 3
- Flexible endpoint materialization
  - Materialize endpoints even with variables
  - One common, simple API for HogQL and insight endpoints regardless of materialization status
