## Q2 2026

### Objective 1: PostHog AI 2.0 migration - <TeamMember name="Georgiy Tarasov" photo />, <TeamMember name="Chris Volzer" photo />

We will migrate PostHog AI to the new sandboxed agent architecture that will be shared by PostHog AI and PostHog Code. This will give PostHog AI code access, and much better support for all of PostHog's products via MCP + skills.

- Move PostHog AI to use the sandbox architecture
- Migrate existing MaxTools to MCP tools and skills
- Evals to compare old vs new PostHog AI

### Objective 2: AI infrastructure - <TeamMember name="Alessandro Pogliaghi" photo />, <TeamMember name="Ryan Sutton" photo />, <TeamMember name="Vojta Bartos" photo />, <TeamMember name="Joshua Snyder" photo />

Provide good infrastructure that teams can build reliable AI features on top of.

- Sandboxes
  - Stable infra to launch sandboxes, snapshot/resume them
  - Better development experience
  - Observability for debugging running tasks
  - Scalability for GA release of PostHog Code
  - Consider support for EC2 sandboxes
  - Run tasks with structured output
- LLM gateway (high priority for internal usage, low priority for productised stuff — we need more bandwidth here)
  - Reliability & fallback providers
  - Support team based usage rather than requiring a personal API key
  - Open source models for internal usage (if we have time)
- MCP server
  - Move from cloudflare to our own infrastructure
  - Improve observability and alerting

### Objective 3: Background agents & automations - <TeamMember name="Alessandro Pogliaghi" photo />, <TeamMember name="Ryan Sutton" photo />, <TeamMember name="Vojta Bartos" photo />, <TeamMember name="Joshua Snyder" photo />

Users are shipping code from the cloud via signals, PostHog Code, and PostHog AI. We want to make this a great experience for them.

- UX of using cloud runs in PostHog Code
- Slack app for kicking off cloud tasks
- Webhooks for auto-fixing CI failures and PR comments
- Agent makes data driven decisions using the MCP server
- Automations for running agents on a schedule
- Snapshot setup and custom environments
- Pricing & billing for cloud runs - <TeamMember name="Annika Schmid" photo />
- Preview environments for PR changes
- Ability to ship PRs from the PostHog AI UI in cloud (if we have time)

### Objective 4: Every product has a first-class AI experience - <TeamMember name="Georgiy Tarasov" photo />, <TeamMember name="Chris Volzer" photo />

The experience of using any product in PostHog via an agent should be amazing. As part of this we want product teams to build three things:

1. **MCP tools**
2. **Skills**
   - Ship a skill for writing skills
   - Help teams understand when their skill is working well or not
   - Track skill usage in our agents, give feedback on traces where we see issues
3. **MCP UI** (second half of Q2)
   - Make a start on using Quill for the AI first experience of each product
   - Give teams a good way to test this in PostHog Code / PostHog AI
