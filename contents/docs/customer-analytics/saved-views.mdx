Saved views let you save column configurations and filters for the [persons](/docs/data/persons) and [groups](/docs/product-analytics/group-analytics) lists. This helps you quickly switch between different ways of viewing your customer data without reconfiguring the table each time.

For example, you might create a "High value customers" view that shows lifetime value, last active date, and subscription status columns with a filter for revenue above a certain threshold.

## What's saved in a view

Each saved view stores:

- **Column configuration**: Which columns are visible and their order.
- **Filters**: Any active filters applied to the list.

Views are saved per table and can be either private or shared with project members.

## Creating a view

To create a saved view:

1. Go to **People & groups** in your PostHog project and select **Persons**
2. Click **Configure columns** in the toolbar above the customer table
3. Add any filters using the **Filter** button
4. Click **Save current view** and enter a name for your view

## Switching views

Click the **view selector dropdown** to see all your saved views. Selecting a view applies its column configuration and filters to the list.

## Updating a view

When you change columns or filters while a saved view is active, an **Update** button appears next to the view selector. Click it to save your changes to the current view.

## Renaming and deleting views

Click the settings icon (gear) next to a view name in the dropdown to access options to **Rename** or **Delete** the view.
