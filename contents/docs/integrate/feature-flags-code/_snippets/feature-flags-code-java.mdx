### Boolean feature flags

`getFeatureFlagResult` returns `null` when the flag doesn't exist.

```java
FeatureFlagResult result = posthog.getFeatureFlagResult("distinct_id_of_your_user", "flag-key");
if (result != null && result.getEnabled()) {
    // Do something differently for this user
}
```

### Multivariate feature flags

```java
FeatureFlagResult result = posthog.getFeatureFlagResult("distinct_id_of_your_user", "flag-key");
if (result == null || result.getVariant() == null) {
    // Flag doesn't exist or has no variant
} else {
    switch (result.getVariant()) {
        case "variant-a":
            // Do something for variant A
            break;
        case "variant-b":
            // Do something for variant B
            break;
    }
}
```

### Payloads

```java
FeatureFlagResult result = posthog.getFeatureFlagResult("distinct_id_of_your_user", "flag-key");
if (result != null && result.getPayload() != null) {
    System.out.println(result.getPayload());

    // Deserialize the payload into a typed object
    MyConfig config = result.getPayloadAs(MyConfig.class);
}
```

### Using an older SDK version?

The `getFeatureFlagResult` API is available in `posthog-server` version **2.3.0** and above. We recommend upgrading to the latest version. Otherwise, use the previous APIs:

```java
// Boolean check
boolean isEnabled = posthog.isFeatureEnabled("distinct_id_of_your_user", "flag-key");

// Multivariate check
Object variant = posthog.getFeatureFlag("distinct_id_of_your_user", "flag-key");

// Fetch the payload
Object payload = posthog.getFeatureFlagPayload("distinct_id_of_your_user", "flag-key");
```
