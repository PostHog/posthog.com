---
title: Using PostHog AI with Product Analytics
---

The [PostHog MCP server](/docs/model-context-protocol) exposes function calling tools to any MCP client, enabling AI agents to interact with PostHog's API via the MCP protocol.

When combining our MCP server with product analytics, your AI agents can query data, build insights, and surface user behavior patterns — all from your code editor or AI client.

For example, with MCP, your agents can:

- **Query trends and metrics**: Ask natural language questions about your analytics data and get results instantly.
- **Run HogQL queries**: Execute complex SQL-style queries against your events, persons, and system tables.
- **Explore funnels and retention**: Understand conversion rates and how users move through your product.
- **Investigate cohorts**: Find and analyze groups of users based on behavior.
- **Build dashboards**: Create and organize insights into dashboards without leaving your editor.

All of this happens directly inside the MCP client, like Cursor, Windsurf, or Claude Code, so you can explore your analytics data without switching context.

Try these example prompts with your MCP-enabled agent:

```
- How many unique users signed up in the last 7 days, broken down by day?
- Show me trends for page views over the last month.
- Create a funnel showing signup to first purchase conversion.
- Write a HogQL query to find users who triggered the signup event but didn't complete onboarding in the last 30 days.
- What cohorts contain users who made a purchase in the last week?
- Which events are most commonly triggered by power users?
```

### HogQL and SQL queries

Your AI agent can execute complex [HogQL](/docs/hogql) queries to analyze your events, persons, and system data.

```
- Write a HogQL query to show the top 10 most common events in the last 7 days.
- Query the events table for users who viewed the pricing page more than 3 times this week.
- Find the median time between signup and first purchase using HogQL.
```

## Using PostHog AI in the app

You can also use [PostHog AI](/docs/posthog-ai/platform-and-chat-ui) directly inside the PostHog app to work with product analytics. Open the AI chat in the sidebar and ask questions like:

```
- What's our signup trend over the last 30 days?
- Build me a dashboard showing key activation metrics.
- Which user properties correlate with higher retention?
```

PostHog AI can create insights, suggest queries, and help you interpret your data — all within the app.

## Install the MCP server

To start querying product analytics with PostHog MCP, install the MCP server in your preferred MCP client.

The easiest way is to use the [AI wizard](/docs/ai-engineering/ai-wizard), which supports setup for agents like Claude, Cursor, Windsurf, VS Code, and more. The wizard guides you through the installation step by step.

```bash
npx @posthog/wizard mcp add
```

You can also install the MCP server by [configuring it manually](/docs/model-context-protocol#manual-install).

Check out the [MCP server documentation](/docs/model-context-protocol) for more detailed instructions and information.
