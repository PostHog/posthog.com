---
title: Using PostHog AI with Surveys
---

[PostHog AI](/docs/posthog-ai) can help you create surveys and analyze responses directly from the chat interface. It has two dedicated survey tools:

- **`analyze_survey_responses`** – Extract themes and actionable insights from your survey response data
- **`create_survey`** – Create new surveys in seconds from a text description

## What you can do

With PostHog AI and Surveys, you can:

- **Analyze survey responses** – Identify common themes, sentiment patterns, and key takeaways from open-ended and structured responses
- **Create surveys** – Describe the survey you want and PostHog AI builds it for you, including question types and targeting
- **Query response data with SQL** – Use PostHog AI to write HogQL queries against your survey response events
- **Correlate responses with behavior** – Combine survey data with product analytics to understand how responses relate to feature adoption, retention, or conversion
- **Find users from responses** – Locate session replays or user profiles for respondents who gave specific answers

## MCP integration

You can also interact with PostHog survey data through the [PostHog MCP server](/docs/model-context-protocol), which enables AI agents in tools like Cursor, Windsurf, and Claude Code to query your PostHog data – including survey responses – using HogQL.

Install the MCP server with the [AI wizard](/docs/ai-engineering/ai-wizard):

```bash
npx @posthog/wizard mcp add
```

See the [MCP server documentation](/docs/model-context-protocol) for setup details.
