## Step two: Enable surveys in your configuration

> **‚ö†Ô∏è Caution:**
> - Surveys are currently in beta and only accessible through the `Experimental` SPI in Swift.
> - We recommend thorough testing in your development environment before deploying to production.
> - Survey APIs may change frequently and introduce breaking changes between SDK releases. Consider this when planning updates.

To enable surveys in your iOS app:

1. First, import PostHog with access to its experimental APIs:
```swift
@_spi(Experimental) import PostHog // Import with Experimental SPI
```

2. Enable surveys in your PostHog configuration:
```swift
let POSTHOG_API_KEY = "<ph_project_api_key>"
// usually 'https://us.i.posthog.com' or 'https://eu.i.posthog.com'
let POSTHOG_HOST = "<ph_client_api_host>"
let config = PostHogConfig(apiKey: POSTHOG_API_KEY, host: POSTHOG_HOST) // TIP: host is optional if you use https://us.i.posthog.com

// Surveys require iOS 15.0 or later
if #available(iOS 15.0, *) {
    config.surveys = true // TIP: If you see a "'surveys' is inaccessible due to '@_spi' protection level" build error, please see step 1 above.
}

PostHogSDK.shared.setup(config)
```
