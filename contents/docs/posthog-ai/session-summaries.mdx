It's hard to find time to watch all your session recordings, but you don't want to miss out on the insights. PostHog AI can summarize your recordings for you, identifying patterns, errors, and identify key summaries for you to review.

## Summarizing session replay recordings

In the [in-app chat](/docs/posthog-ai/platform-and-chat-ui#in-app-chat), you can ask PostHog AI to summarize your session recordings. PostHog AI will pick up on the filters you've applied to the session recordings and use them to summarize the recordings.

PostHog AI will categorize the recordings by behavior patterns and generate a summary report for you to review. You can always ask PostHog AI more questions about the session recordings, and to identify specific recordings for you to review.

While summarizing, PostHog AI displays real-time progress so you can track what's happening. You'll see which phase the summarization is in (fetching data, watching sessions, searching for patterns, or building the report), a progress bar with per-session status, and an ETA countdown during the watching phase.

## Searching for specific recordings

You can also ask PostHog AI to search for specific recordings. For example, you can ask PostHog AI to find all the recordings where a user rage clicked or used the navigation menu.

Start by clicking the <span className="inline-block"><IconSparkles className="h-4 w-4" /> </span> icon next to the session recording filters.

## Summarize session replays in Experiments

You can also summarize session replays directly from the [Experiments](/docs/experiments) interface. This helps you understand how users in different experiment variants behave.

To use this feature:

1. Go to an experiment that has started and has results
2. Click the **Summarize session replays** button
3. PostHog AI opens and automatically analyzes user behavior patterns from session replays across your experiment variants

PostHog AI categorizes recordings by variant and identifies behavioral differences, helping you understand not just _what_ happened in your experiment, but _why_ users responded differently to each variant.

## Customize summaries with custom tags

By default, PostHog AI uses a fixed set of tags (like `onboarding`, `error`, `frustration`) to categorize session recordings. These work well for general use, but they may not capture patterns specific to your product.

Custom tags let you define your own taxonomy so the AI can identify product-specific behaviors. For example, a checkout flow might need a `cart_abandoned` tag, while a B2B tool might benefit from `workspace_invite`.

To configure custom tags:

1. Go to [**Settings** > **Environment** > **Session replay**](https://app.posthog.com/settings/environment-replay#replay-ai-config)
2. Scroll to the **AI product context** section
3. Under **Custom tags**, add tags with a `snake_case` name and a description

Descriptions tell the AI when to apply each tag, so be specific. For example, a tag named `pricing_hesitation` might have the description "User repeatedly visits the pricing page or toggles between plans without converting."

You can add up to 15 custom tags per project. Custom tags are kept separate from built-in tags so they don't collide in the UI. Only admins can configure custom tags.
