The Sentry connector can link error tracking and project data from your Sentry organization to PostHog.

| Dataset | Description |
|---|---|
| projects | Projects in your organization |
| teams | Teams in your organization |
| members | Organization members |
| releases | Releases across your organization |
| environments | Deployment environments |
| monitors | Cron monitors |
| issues | Error issues (supports incremental sync) |
| project_events | Error events per project (supports incremental sync) |
| project_users | Users affected by errors per project |
| project_client_keys | DSN keys per project |
| project_service_hooks | Webhooks per project |
| issue_events | Events per issue (supports incremental sync) |
| issue_hashes | Stack trace hashes per issue |
| issue_tag_values | Tag values per issue (supports incremental sync) |

## Creating a Sentry auth token

To connect Sentry to PostHog, you need an internal integration token:

1. In Sentry, go to **Settings** > **Developer Settings** > [**Custom Integrations**](https://sentry.io/settings/custom-integrations/).
2. Click **Create New Integration** and select **Internal Integration**.
3. Give your integration a name (e.g., "PostHog Data Sync").
4. Under **Permissions**, grant **Read** access to: Project, Team, Member, Organization, Release, and Issue & Event.
5. Click **Save Changes**.
6. Copy the **Token** value – you'll need this when linking the source in PostHog.

## Linking Sentry as a source

1. Go to the [sources tab](https://app.posthog.com/data-management/sources) of the data pipeline section in PostHog.
2. Click **+ New source** and then click **Link** next to Sentry.
3. Enter your **Organization slug** (found in your Sentry URL: `https://sentry.io/organizations/{organization_slug}/`).
4. Paste the **Auth token** you created above.
5. *Optional:* If you're using a regional Sentry instance (US or EU), select your **API base URL** (`https://us.sentry.io` or `https://de.sentry.io`).
6. *Optional:* Add a prefix to your table names.
7. Click **Next**.

The data warehouse then starts syncing your Sentry data. You can see details and progress in the [data pipeline sources tab](https://app.posthog.com/data-management/sources).

> The `issues` and `issue_tag_values` tables support incremental syncing using the `lastSeen` field. The `project_events` and `issue_events` tables support incremental syncing using the `dateCreated` field. Other tables use full refresh syncs.
