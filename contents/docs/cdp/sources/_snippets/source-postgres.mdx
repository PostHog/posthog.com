The Postgres connector can link your database tables to PostHog. There are two access methods:

- **Sync to warehouse** (default): Syncs selected tables into PostHog-managed storage for querying. You can join this data with your PostHog data.
- **Query directly** (beta): Runs queries live against your Postgres connection without copying data. Data from this source can't be joined with PostHog data.

## Sync to warehouse

To sync Postgres data into PostHog:

1. Go to the [Data pipeline page](https://app.posthog.com/data-management/sources) and the sources tab in PostHog
2. Click **New source** and select Postgres
3. Select **Sync to warehouse** as the access method
4. Enter your database connection details:
    - **Host:** The hostname or IP of your database server like `db.example.com` or `123.132.1.100`.
    - **Port:** The port your database server is listening on. The default is `5432`.
    - **Database:** The name of the database you want like `analytics_db`.
    - **User:** The username with the necessary permissions to access the database.
    - **Password:** The password for the user.
    - **Schema:** The schema for your database where your tables are located. The default is `public`.
5. Optionally set a **Table prefix** and **Description** for the source
6. Select the tables and sync method for each, then click **Import**

The data warehouse then starts syncing your Postgres data. You can see details and progress in the [sources tab](https://app.posthog.com/data-management/sources).

<CalloutBox icon="IconWarning" title="SSL/TLS required for new sources" type="caution">

PostgreSQL sources created after February 18, 2026 require SSL/TLS encryption for database connections. Ensure your PostgreSQL database supports SSL/TLS connections before linking. Existing sources created before this date are not affected.

</CalloutBox>

> **Permissions** The Postgres source only requires read permissions on the schemas and tables you intend to sync.

## Query directly

<CalloutBox icon="IconFlag" title="Beta" type="info">

Direct query access for Postgres is in beta. This feature is being actively developed.

</CalloutBox>

The direct query method lets you run queries against your Postgres database without syncing data into PostHog. This is useful when you want to query live data or avoid duplicating large tables.

To set up a direct Postgres connection:

1. Go to the [Data pipeline page](https://app.posthog.com/data-management/sources) and the sources tab in PostHog
2. Click **New source** and select Postgres
3. Select **Query directly** as the access method
4. Enter a **Name** for the connection (shown in the query editor when selecting this Postgres connection)
5. Enter your database connection details (host, port, database name, user, and password)
6. Select which tables should be available for querying, then click **Save tables**

Your enabled tables are immediately available in the SQL editor.

> **Note:** Direct query sources don't support SSH tunnels. Data from direct query sources can't be joined with PostHog data.
