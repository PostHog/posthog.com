---
title: Using PostHog AI to analyze your LLM data
---

[PostHog AI](/docs/posthog-ai) can query and analyze the LLM data you're capturing with [LLM Analytics](/docs/llm-analytics). Ask questions in natural language and get answers based on your `$ai_generation` events, traces, and spans.

## What you can do

- **Query generation data with SQL** – PostHog AI writes and runs HogQL queries against your LLM events so you don't have to.
- **Analyze token usage and costs** – Break down [input and output tokens](/docs/llm-analytics/generations) and [costs](/docs/llm-analytics/calculating-costs) across models and providers.
- **Find slow or expensive generations** – Identify generations with high latency or cost to optimize performance.
- **Identify error patterns** – Surface [error trends](/docs/llm-analytics/errors) in your LLM calls by model, provider, or time period.
- **Analyze user feedback** – Explore [user feedback](/docs/llm-analytics/collect-user-feedback) on AI responses to find quality issues.

For more prompt ideas across all PostHog products, see the [PostHog AI example prompts](/docs/posthog-ai/example-prompts) page.

## Use with MCP

You can also analyze your LLM data through the [PostHog MCP server](/docs/model-context-protocol). This lets AI agents in tools like Cursor, Claude Code, and Windsurf query your LLM Analytics data using HogQL – directly from your code editor.

Install the MCP server with:

```bash
npx @posthog/wizard mcp add
```

See the [MCP server documentation](/docs/model-context-protocol) for setup instructions.
