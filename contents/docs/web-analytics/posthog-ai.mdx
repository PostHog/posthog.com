---
title: PostHog AI
---

[PostHog AI](/docs/posthog-ai) can help you analyze web traffic, investigate anomalies, and build insights using natural language – directly from the PostHog app.

Open PostHog AI from any page by clicking the sparkle icon in the corner. It has full access to your web analytics data, including pageviews, sessions, referral sources, UTM parameters, and more.

## Analyze web traffic patterns

Ask PostHog AI to surface trends, compare time periods, or break down traffic by any dimension.

```
Show me a breakdown of unique visitors by page over the last 30 days.
```

```
Compare this week's traffic to last week. What changed?
```

```
Which pages had the biggest drop in traffic this month?
```

## Understand referral sources and UTM campaigns

PostHog AI can query your referral and UTM data to help you understand where traffic comes from and how campaigns perform.

```
Break down my traffic by referral source over the last 7 days.
```

```
Which UTM campaigns drove the most conversions this quarter?
```

```
Show me traffic from organic search vs paid ads over the last 90 days.
```

## Investigate conversion funnels

Use PostHog AI to build and analyze conversion funnels from your web analytics data.

```
Create a funnel from landing page view to signup over the last 30 days.
```

```
What's my conversion rate from the pricing page to checkout?
```

```
Which referral sources have the highest conversion rates?
```

## Investigate traffic anomalies

PostHog AI can help you dig into unexpected spikes or drops in traffic. For a full walkthrough of this workflow, see [Investigate web traffic with PostHog AI](/docs/posthog-ai/investigate-web-traffic).

```
There's a traffic spike on March 1st. Create a dashboard with breakdowns by referral, country, and user agent to investigate.
```

```
Find session replays from users who visited the homepage from an unusual referral source.
```

## Use external AI agents with MCP

You can also connect external AI agents – like Cursor, Claude Code, or Windsurf – to your PostHog data using the [PostHog MCP server](/docs/model-context-protocol). This lets you query web analytics data from your code editor or other MCP clients.
