The **SupportHog** Teams bot connects your Microsoft Teams workspace to PostHog Support, enabling two-way sync between Teams conversations and support tickets.

- **Teams to PostHog** – messages in Teams automatically create support tickets and add comments
- **PostHog to Teams** – team replies in PostHog are posted back to the Teams thread

## Connecting Microsoft Teams

1. Go to **Support > Settings** in PostHog
2. Click **Connect Microsoft Teams**
3. Sign in with your Microsoft account and authorize the SupportHog app (requires Teams admin permissions)
4. Select a **Teams group** that contains your support channel
5. The SupportHog app is installed into the selected group
6. Select a **support channel** – messages posted there automatically become tickets

<CalloutBox icon="IconInfo" title="One tenant per project" type="fyi">

Each PostHog project can connect to one Microsoft Teams tenant. If the tenant is already connected to another project, you'll see an error.

</CalloutBox>

### App catalog requirement

If SupportHog isn't available in your Microsoft tenant's app catalog, your organization's Teams admin needs to upload the SupportHog app package to the [org catalog](https://admin.teams.microsoft.com/policies/manage-apps) first. This is a one-time setup. Once uploaded, click **Retry install** in PostHog settings.

## Configuration

After connecting, configure the integration in **Support > Settings**:

| Setting             | Description                                                                                       |
| ------------------- | ------------------------------------------------------------------------------------------------- |
| **Teams group**     | The Microsoft Teams group that contains your support channel.                                     |
| **Support channel** | Every top-level message in this channel becomes a ticket. Thread replies sync as ticket messages. |
| **Bot mentions**    | Always active – `@SupportHog` in any channel creates a ticket from that message.                  |

### Picking a support channel

When you select a Teams group, PostHog lists the channels in that group. Select the channel you want to use as your dedicated support inbox. You can click **Refresh** to reload the channel list if new channels have been added.

## How tickets are created

There are two ways a Teams message becomes a support ticket:

### 1. Support channel messages

Any top-level message in your configured support channel automatically creates a new ticket. Thread replies on that message sync as ticket messages.

### 2. Bot mentions

Type `@SupportHog` in any channel to create a ticket from that message. If mentioned inside an existing ticket thread, the message is added as a comment instead.

Both methods:

- Set `channel_source` to `teams` on the ticket
- Resolve the Teams user's name and email via Microsoft Graph
- Convert Teams message formatting to PostHog's rich text format
- Post a confirmation Adaptive Card in the Teams thread with the ticket number and a link to view it in PostHog

## Two-way sync

### Teams to PostHog

- New messages in the support channel create tickets
- Thread replies sync as ticket messages
- Bot mentions in any channel create tickets

### PostHog to Teams

- When a team member replies to a Teams-sourced ticket in PostHog, the reply is posted back to the original Teams thread
- The reply shows the team member's name
- Rich text formatting (bold, italic, links, lists, code) is preserved

**Private notes** (internal notes) are never sent to Teams.

## Viewing Teams tickets

Teams-sourced tickets appear in your [inbox](/docs/support/inbox) alongside widget, email, and Slack tickets. Each Teams ticket shows:

- A Teams icon indicating the channel source
- The Teams user's name on messages

You can filter the ticket list by channel source to see only Teams tickets.

## Disconnecting

To remove the Teams integration:

1. Go to **Support > Settings**
2. Click **Disconnect Microsoft Teams**
3. Confirm the disconnection

Existing tickets created from Teams are not affected – they remain in your inbox but will no longer sync new messages.
