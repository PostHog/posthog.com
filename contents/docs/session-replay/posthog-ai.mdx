[PostHog AI](/docs/posthog-ai) can help you get more out of Session Replay without manually watching every recording. It can search recordings with natural language, summarize sessions to surface patterns and issues, and help you triage large volumes of replays.

## Search recordings with natural language

Instead of manually configuring filters, you can ask PostHog AI to find recordings for you. Click the <span className="inline-block"><IconSparkles className="h-4 w-4" /> </span> icon next to the session recording filters, or open [PostHog AI](https://app.posthog.com/#panel=max) from anywhere in the app.

PostHog AI uses the `search_session_recordings` tool to translate your natural language query into the right filters and return matching recordings.

Try prompts like:

```
Show me recordings where users rage clicked in the last 7 days.
```

```
Find sessions where users visited the pricing page but didn't sign up.
```

```
Show recordings of users who encountered errors on the checkout page.
```

## Summarize sessions

Watching every recording isn't realistic. PostHog AI can summarize your recordings for you using the `session_summarization` tool, identifying behavior patterns, errors, and key moments worth reviewing.

PostHog AI picks up on any filters you've applied to the recordings list and uses them to focus its summary. It categorizes recordings by behavior patterns and generates a summary report.

## Summarize replays in Experiments

You can also summarize session replays directly from the [Experiments](/docs/experiments) interface. This helps you understand how users in different experiment variants behave.

1. Go to an experiment that has started and has results.
2. Click **Summarize session replays**.
3. PostHog AI analyzes user behavior patterns from session replays across your experiment variants.

PostHog AI categorizes recordings by variant and identifies behavioral differences, helping you understand not just *what* happened in your experiment, but *why* users responded differently to each variant.

Here are more prompts to try with PostHog AI for Session Replay:

## Use MCP for external AI agents

You can also interact with Session Replay data through external AI agents using the [PostHog MCP server](/docs/model-context-protocol). This enables tools like Cursor, Windsurf, or Claude Code to search and analyze recordings on your behalf.

Install the MCP server with:

```bash
npx @posthog/wizard mcp add
```

See the [MCP server documentation](/docs/model-context-protocol) for setup instructions.
