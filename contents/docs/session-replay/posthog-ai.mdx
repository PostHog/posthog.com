PostHog AI helps you find, filter, and understand session recordings without manually watching hours of replays. You can use it both in the PostHog app and via the [MCP server](/docs/model-context-protocol) in your code editor.

## Finding recordings with AI

On the [Session Replay](https://us.posthog.com/replay) page, click the <span className="inline-block"><IconSparkles className="h-4 w-4" /> </span> icon next to the recording filters to ask PostHog AI to find specific recordings for you.

## Summarizing recordings

PostHog AI can summarize your recordings to surface user behavior patterns, errors, and key moments — so you don't have to watch every replay yourself. In the [in-app chat](/docs/posthog-ai/chat), ask PostHog AI to summarize recordings. It picks up on any filters you've applied and categorizes recordings by behavior patterns.

## Querying recordings via MCP

The [PostHog MCP server](/docs/model-context-protocol) lets AI agents in tools like Cursor, Claude Code, and VS Code query recording metadata and events directly. This is useful for investigating user behavior without leaving your code editor.

For example, your agent can:

- **Find recordings by event**: Query for sessions containing specific events like errors, button clicks, or page views.
- **Filter by user properties**: Find recordings from users matching cohorts, regions, or account types.
- **Analyze session metadata**: Look at session durations, activity levels, and event counts.

### Install the MCP server

The easiest way to get started is with the [AI wizard](/docs/ai-engineering/ai-wizard):

```bash
npx @posthog/wizard mcp add
```

You can also [configure it manually](/docs/model-context-protocol#manual-install). See the [MCP server documentation](/docs/model-context-protocol) for full setup instructions.

### Example prompts

```
- Show me sessions where users hit an error on the /checkout page this week.
- Find recordings from users in the enterprise plan cohort.
- Which sessions had the most events in the last 7 days?
- Query session replays for users who triggered the signup event but didn't complete onboarding.
```
