> **Note:** Session replay for Unity is currently **experimental**. The Unity SDK is in beta. Please report any issues on [GitHub](https://github.com/PostHog/posthog-unity/issues).

## Step 1: Install the SDK

Requires **Unity 2021.3 LTS** or later with **.NET Standard 2.1** API compatibility level.

In the Unity Editor, open **Window > Package Manager**, click the **+** button, select **Add package from git URL**, and enter:

```
https://github.com/PostHog/posthog-unity.git?path=com.posthog.unity
```

See the [Unity SDK docs](/docs/libraries/unity) for full setup instructions.

## Step 2: Enable session replay

Add `SessionReplay = true` to your PostHog configuration:

```csharp
using PostHogUnity;

PostHog.Setup(new PostHogConfig
{
    ApiKey = "<ph_project_token>",
    Host = "<ph_client_api_host>",
    SessionReplay = true
});
```

If you use the Unity Inspector for configuration, enable the **Session Replay** toggle in **Edit > Project Settings > PostHog**.

## Step 3: Configure session replay (optional)

Customize capture behavior with `SessionReplayConfig`:

```csharp
using PostHogUnity;
using PostHogUnity.SessionReplay;

PostHog.Setup(new PostHogConfig
{
    ApiKey = "<ph_project_token>",
    Host = "<ph_client_api_host>",
    SessionReplay = true,
    SessionReplayConfig = new PostHogSessionReplayConfig
    {
        // Screenshot settings
        ThrottleDelaySeconds = 1.0f,    // Min seconds between captures (default: 1.0)
        ScreenshotQuality = 80,         // JPEG quality 1-100 (default: 80)
        ScreenshotScale = 0.75f,        // Resolution scale 0.1-1.0 (default: 0.75)

        // Telemetry capture
        CaptureNetworkTelemetry = true, // Record HTTP request metadata (default: true)
        CaptureLogs = false,            // Record console logs (default: false)
        MinLogLevel = SessionReplayLogLevel.Error, // Log, Warning, or Error (default: Error)

        // Queue settings
        FlushAt = 20,                   // Events before auto-flush (default: 20)
        FlushIntervalSeconds = 30,      // Seconds between flushes (default: 30)
        MaxQueueSize = 100              // Max queued events (default: 100)
    }
});
```

## How it works

The Unity SDK captures screenshots using `AsyncGPUReadback` to avoid blocking the main thread. It also records touch/mouse input, network requests and console logs alongside each screenshot.

Unlike web session replay which records DOM changes, Unity session replay captures periodic screenshots of your game's rendered output. This means replay fidelity depends on your capture settings (frequency, resolution and quality).

## Platform support

| Platform | Session replay support |
|----------|----------------------|
| Windows/Mac/Linux | Full |
| iOS | Full |
| Android | Full |
| WebGL | Not supported |

Session replay requires `AsyncGPUReadback` support. On WebGL, session replay is automatically disabled.

## Performance tips

Session replay captures screenshots which can impact performance. To optimize:

- **Increase `ThrottleDelaySeconds`** to reduce capture frequency
- **Lower `ScreenshotScale`** to reduce resolution (e.g., `0.5f`)
- **Lower `ScreenshotQuality`** to reduce file size (e.g., `50`)
- **Disable `CaptureNetworkTelemetry` and `CaptureLogs`** if not needed
