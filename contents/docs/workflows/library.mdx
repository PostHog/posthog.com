## Workflow templates

You can save workflows as reusable templates to share across your team. When saving a workflow as a template, select a scope to control visibility:

| Scope | Description |
|-------|-------------|
| **Project** | Template is only visible within the current project (default) |
| **Organization** | Template is visible to all projects within your organization |
| **Global** | Official PostHog templates (staff-only) |

Organization-scoped templates are accessible to all teams within the same organization. Any team member can use and update these shared templates. Templates remain isolated to their organization and aren't visible to other organizations.

### Built-in templates

PostHog includes a set of pre-built workflow templates to help you get started quickly. These are available when you click **New workflow** and cover common use cases like onboarding, engagement, and alerting:

| Template | Description |
|----------|-------------|
| **Celebrate milestone** | Send congrats when users reach an important product milestone |
| **Feature adoption tips** | Guide users with a drip campaign after they adopt a feature |
| **Re-engagement workflow** | Detect inactive users and remind them to come back |
| **Trial ending reminder** | Remind users before their trial expires so they can upgrade |
| **Heavy usage detected** | Notify users when they're close to their usage limit |
| **Negative survey response** | Alert your team in Slack when a user submits negative feedback |
| **User stuck in onboarding** | Alert your team when users start but don't complete onboarding |
| **Support ticket notification** | Notify your support team in Slack when a new ticket is created |
| **Unresolved ticket alert** | Escalate support tickets that remain unresolved after a set time |
| **High intent notification** | Alert your sales team when a user shows strong buying intent |
| **Repeated failure alert** | Alert your team when a user action fails repeatedly |
| **Unlocking advanced features** | Unlock features and notify users when they complete a key action |
| **Unused features education** | Send tips to users who haven't tried an important feature yet |
| **User upgrade webhook** | Send a webhook to external systems when a user upgrades |

Each template includes pre-configured triggers, delays, and dispatch steps that you can customize for your needs.

## Liquid and merge tags

All template types support [Liquid templating](https://shopify.github.io/liquid/) to personalize your messages with user properties and data. You can type liquid directly, or use the merge tag dropdown in your message templates, which will show you the entire list of available person properties (see email example below).

<CalloutBox icon="IconInfo" title="Tip: Use default values with Liquid" type="fyi">

When using Liquid with existing user properties, you should include a default value if the property could be null or may not exist. For example:

```liquid
{{ person.name | default: "there" }}
```

This ensures your message displays correctly even when a property is missing.

</CalloutBox>

### Email merge tags

In addition to person properties, PostHog provides these system merge tags for email templates:

| Merge tag | Description |
|-----------|-------------|
| `{{unsubscribe_url}}` | Links to a page where recipients can manage their email preferences |
| `{{unsubscribe_url_one_click}}` | Enables RFC 8058-compliant one-click unsubscribe. When email clients send a POST request to this URL, recipients are automatically unsubscribed from all marketing emails. When clicked directly, it displays an unsubscribe confirmation page. |

Use the one-click unsubscribe URL to improve email deliverability and comply with email regulations like CAN-SPAM. Most modern email clients display a prominent unsubscribe button when they detect proper one-click unsubscribe headers.

The builder allows you to:

- Drag different types of content into the email like columns, buttons, text etc.
- Format the content with blocks, breaking up content into multiple sections for example

### Using Liquid in email templates

You can type Liquid syntax directly into your email template. We also provide a merge tags dropdown that you can click to see all available user properties and easily insert them into your template.

<ProductScreenshot
    imageLight="https://res.cloudinary.com/dmukukwp6/image/upload/q_auto,f_auto/merge_tags_6c859a905f.png"

If you type Liquid syntax directly and the Liquid doesn't compile correctly, the email won't be sent. Always verify your Liquid syntax is valid before starting a workflow.

### Testing templates

It's best practice to test a message in a workflow before starting the workflow. This helps ensure your template displays correctly with real user data and that all Liquid variables resolve properly.
