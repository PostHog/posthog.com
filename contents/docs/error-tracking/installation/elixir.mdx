<Steps>

<Step title="Install the Elixir SDK" badge="required">

Add the [PostHog Elixir SDK](/docs/libraries/elixir) to your list of dependencies in `mix.exs`:

```elixir
def deps do
  [
    {:posthog, "~> 2.2.0"}
  ]
end
```

Then fetch the dependencies:

```bash
mix deps.get
```

</Step>

<Step title="Configure PostHog" badge="required">

Add your PostHog configuration to `config/config.exs`:

```elixir
config :posthog,
  api_key: "<ph_project_token>",
  api_host: "<ph_client_api_host>"
```

You can see all the available configuration options in the [PostHog.Config](https://hexdocs.pm/posthog/PostHog.Config.html) module.

</Step>

<Step title="Error tracking is enabled by default" badge="info">

The Elixir SDK automatically captures exceptions when error tracking is enabled. Since it's built on top of Elixir's `Logger` module, it automatically captures any `Logger.error` calls as exceptions.

<CalloutBox icon="IconInfo" title="Automatic capture" type="fyi">

Error tracking is enabled by default in the Elixir SDK. Exceptions thrown by your application are automatically captured and sent to PostHog.

</CalloutBox>

</Step>

<Step title="Disable error tracking (optional)" badge="optional">

If you want to disable automatic error tracking, set `enable_error_tracking` to `false` in your configuration:

```elixir
config :posthog,
  api_key: "<ph_project_token>",
  api_host: "<ph_client_api_host>",
  enable_error_tracking: false
```

</Step>

<Step title="Verify error tracking" badge="recommended">

Trigger a test exception to confirm events are being sent to PostHog. You should see them appear in the [activity feed](https://us.posthog.com/activity/explore).

```elixir
require Logger

# Trigger a test error
Logger.error("This is a test error from Elixir")
```

Or raise an exception in your application:

```elixir
raise "This is a test exception"
```

</Step>

</Steps>
