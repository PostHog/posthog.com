<Step title="Install the .NET SDK" badge="required">

Install the [PostHog .NET SDK](/docs/libraries/dotnet):

<CodeTab labels={["ASP.NET Core", "Console / Worker"]}>

```bash
dotnet add package PostHog.AspNetCore
```

```bash
dotnet add package PostHog
```

</CodeTab>

Use `PostHog.AspNetCore` for ASP.NET Core applications. For console apps, workers, or other .NET projects, use the base `PostHog` package.

</Step>

<Step title="Configure PostHog" badge="required">

**For ASP.NET Core applications**, add PostHog to your services in `Program.cs`:

```csharp
using PostHog;

var builder = WebApplication.CreateBuilder(args);
builder.AddPostHog();
```

Then configure your project token and host in `appsettings.json`:

```json
{
  "PostHog": {
    "ProjectToken": "<ph_project_token>",
    "HostUrl": "<ph_client_api_host>"
  }
}
```

**For console or worker applications**, configure the client directly:

```csharp
using PostHog;

var services = new ServiceCollection();
services.AddPostHog(options => {
    options.ProjectToken = "<ph_project_token>";
    options.HostUrl = new Uri("<ph_client_api_host>");
});
var serviceProvider = services.BuildServiceProvider();
var posthog = serviceProvider.GetRequiredService<IPostHogClient>();
```

You can find your project token and instance address in your [project settings](https://app.posthog.com/settings/project).

</Step>

<Step title="Capture exceptions" badge="required">

Use `CaptureException` to send handled exceptions to PostHog as `$exception` events with stack traces.

### Basic usage

```csharp
try
{
    // Code that might throw
    ProcessOrder(orderId);
}
catch (Exception ex)
{
    posthog.CaptureException(ex, "user_distinct_id");
    throw; // Re-throw if needed
}
```

### With additional properties

Include extra context with your exceptions:

```csharp
try
{
    ProcessCheckout(orderId);
}
catch (Exception ex)
{
    posthog.CaptureException(
        ex,
        "user_distinct_id",
        new Dictionary<string, object>
        {
            ["order_id"] = orderId,
            ["$current_url"] = "/checkout",
        }
    );
}
```

<CalloutBox icon="IconInfo" title="Manual capture only" type="fyi">

The .NET SDK supports manual exception capture only. Automatic capture of unhandled exceptions isn't available yet. For ASP.NET Core apps, add a global exception handler middleware that calls `CaptureException`.

</CalloutBox>

</Step>

<Step title="Add global exception handling (ASP.NET Core)" badge="recommended">

For ASP.NET Core apps, add middleware to capture unhandled exceptions:

```csharp
using PostHog;

var app = builder.Build();

app.Use(async (context, next) =>
{
    try
    {
        await next(context);
    }
    catch (Exception ex)
    {
        var posthog = context.RequestServices
            .GetRequiredService<IPostHogClient>();
        
        posthog.CaptureException(
            ex,
            context.User.Identity?.Name ?? "anonymous",
            new Dictionary<string, object>
            {
                ["$current_url"] = context.Request.Path.ToString(),
                ["$request_method"] = context.Request.Method,
            }
        );
        
        throw;
    }
});
```

Add this middleware early in the pipeline to catch exceptions from all downstream middleware and endpoints.

</Step>

<Step checkpoint title="Verify error tracking" badge="recommended">

Trigger a test exception to confirm events are being sent to PostHog. You should see them appear in the [Error Tracking](https://app.posthog.com/error_tracking) tab.

```csharp
try
{
    throw new InvalidOperationException("Test exception from .NET");
}
catch (Exception ex)
{
    posthog.CaptureException(ex, "test_user");
}

// Ensure the event is sent before the process exits
await posthog.FlushAsync();
```

</Step>

import { Steps, Step } from 'components/Docs/Steps'
import { CalloutBox } from 'components/Docs/CalloutBox'

<Steps>

<Step title="Install the .NET SDK" badge="required">

For ASP.NET Core projects, install the [PostHog.AspNetCore](/docs/libraries/dotnet) package:

```bash
dotnet add package PostHog.AspNetCore
```

For other .NET projects, such as console applications, MAUI, or Blazor, install the core `PostHog` package:

```bash
dotnet add package PostHog
```

</Step>

<Step title="Configure PostHog" badge="required">

For ASP.NET Core, add PostHog to your `Program.cs` file:

```csharp
using PostHog;

var builder = WebApplication.CreateBuilder(args);

builder.AddPostHog();
```

Configure your project token and host in `appsettings.json`:

```json
{
  "PostHog": {
    "ProjectToken": "<ph_project_token>",
    "HostUrl": "<ph_client_api_host>"
  }
}
```

For non-ASP.NET Core projects, initialize the core client as a singleton:

```csharp
using PostHog;

public static readonly PostHogClient PostHog = new(new PostHogOptions {
    ProjectToken = "<ph_project_token>",
    HostUrl = new Uri("<ph_client_api_host>"),
});
```

You can find your project token and instance address in the [project settings](https://app.posthog.com/project/settings) page in PostHog.

</Step>

<Step title="Capture exceptions" badge="required">

The .NET SDK supports manual exception capture with `CaptureException`. It builds the `$exception` event for you, including exception type, message, stack frames, inner exceptions, aggregate exceptions, source context when available, and .NET runtime metadata.

The examples below assume `posthog` is an `IPostHogClient` from dependency injection, or your singleton `PostHogClient`.

```csharp
try
{
    ProcessOrder(orderId);
}
catch (Exception exception)
{
    posthog.CaptureException(exception, "user_distinct_id");
}
```

Pass additional properties to add request, tenant, or domain context to the exception:

```csharp
try
{
    ProcessOrder(orderId);
}
catch (Exception exception)
{
    posthog.CaptureException(
        exception,
        "user_distinct_id",
        new Dictionary<string, object>
        {
            ["order_id"] = orderId,
            ["environment"] = "production",
        }
    );
}
```

You can also attach feature flag values to the exception event with a feature flag snapshot:

```csharp
var flags = await posthog.EvaluateFlagsAsync("user_distinct_id");

posthog.CaptureException(
    exception,
    "user_distinct_id",
    properties: null,
    groups: null,
    flags: flags
);
```

<CalloutBox icon="IconInfo" title="Automatic capture" type="fyi">

Automatic exception capture is not available in the .NET SDK yet. Wrap the code paths you want to monitor and call `CaptureException` from your exception handlers.

</CalloutBox>

</Step>

<Step title="Capture ASP.NET Core request exceptions" badge="recommended">

For ASP.NET Core apps, add middleware to capture unhandled request exceptions and then rethrow them so your existing error handling still runs:

```csharp
using Microsoft.Extensions.DependencyInjection;
using PostHog;

app.Use(async (context, next) =>
{
    try
    {
        await next();
    }
    catch (Exception exception)
    {
        var posthog = context.RequestServices.GetRequiredService<IPostHogClient>();
        var distinctId = context.User.Identity?.Name ?? context.TraceIdentifier;

        posthog.CaptureException(
            exception,
            distinctId,
            new Dictionary<string, object>
            {
                ["$current_url"] = $"{context.Request.Scheme}://{context.Request.Host}{context.Request.Path}",
                ["$request_method"] = context.Request.Method,
                ["$pathname"] = context.Request.Path.ToString(),
            }
        );

        throw;
    }
});
```

</Step>

<Step checkpoint title="Verify error tracking" badge="recommended">

Trigger and capture a test exception to confirm events are being sent to PostHog. You should see it appear in the [Error Tracking](https://app.posthog.com/error_tracking) tab.

```csharp
try
{
    throw new InvalidOperationException("Test exception from .NET");
}
catch (Exception exception)
{
    posthog.CaptureException(exception, "test_user");
    await posthog.FlushAsync();
}
```

For serverless environments, call `FlushAsync()` before the process exits so queued exception events are sent.

</Step>

</Steps>
