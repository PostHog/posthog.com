<CalloutBox icon="IconInfo" title="Manual capture only" type="fyi">

The .NET SDK supports manual exception capture with `CaptureException`. Automatic exception capture is not available yet.

</CalloutBox>

<Steps>

<Step title="Install the .NET SDK" badge="required">

Install the [PostHog .NET SDK](/docs/libraries/dotnet) package for your app:

```bash file=ASP.NET Core
dotnet add package PostHog.AspNetCore
```

```bash file=Console or worker
dotnet add package PostHog
```

</Step>

<Step title="Configure PostHog" badge="required">

Register PostHog and set your project token and host.

For **ASP.NET Core** apps, call `AddPostHog()` on the builder:

```csharp file=Program.cs
using PostHog;

var builder = WebApplication.CreateBuilder(args);
builder.AddPostHog();
```

Then add your project token and host to `appsettings.json`:

```json file=appsettings.json
{
  "PostHog": {
    "ProjectToken": "<ph_project_token>",
    "HostUrl": "<ph_client_api_host>"
  }
}
```

You can find your project token and instance address in the [project settings](https://app.posthog.com/settings/project) page.

</Step>

<Step title="Capture exceptions" badge="required">

Use `CaptureException` to manually capture handled exceptions. This sends a `$exception` event with stack frames, inner exceptions, aggregate exceptions, source context when available, and .NET runtime metadata.

### Basic usage

```csharp
try
{
    ProcessOrder(orderId);
}
catch (Exception exception)
{
    posthog.CaptureException(exception, "user_distinct_id");
    throw;
}
```

### With additional properties

Include custom properties to add request, tenant, or domain context:

```csharp
try
{
    ProcessOrder(orderId);
}
catch (Exception exception)
{
    posthog.CaptureException(
        exception,
        "user_distinct_id",
        new Dictionary<string, object>
        {
            ["order_id"] = orderId,
            ["environment"] = "production",
        }
    );
    throw;
}
```

</Step>

<Step checkpoint title="Verify error tracking" badge="recommended">

Trigger a test exception to confirm events are being sent to PostHog. You should see them appear in the [Error Tracking](https://app.posthog.com/error_tracking) tab.

```csharp
try
{
    throw new Exception("Test exception from .NET");
}
catch (Exception exception)
{
    posthog.CaptureException(exception, "test_user");
}
```

</Step>

</Steps>
