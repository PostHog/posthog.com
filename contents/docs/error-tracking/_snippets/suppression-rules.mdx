Suppress exceptions before they create issues by [configuring suppression rules](https://app.posthog.com/error_tracking?activeTab=configuration#selectedSetting=error-tracking-suppression-rules). Rules are evaluated both client-side and server-side, dropping matching exceptions before they're processed into issues.

You can filter on exception type and message. Server-side evaluation happens after stack trace symbolication, so you have access to resolved (unminified) exception data. Rules using negative operators (like "is not" or "does not contain") are only evaluated server-side because client-side data may still be minified.

You can also set a **sampling rate** (0.0 to 1.0) on a rule to suppress only a percentage of matching exceptions. For example, a sampling rate of `0.5` suppresses 50% of matches, letting you reduce noise from high-volume errors while still capturing some occurrences for monitoring.
