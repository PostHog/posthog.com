Set up alerts to stay informed when issues need attention. PostHog supports three alert triggers:

- **Issue created** - Get notified when a new error issue is detected
- **Issue reopened** - Get notified when a previously resolved issue reoccurs
- **Issue spiking** - Get notified when an issue's error rate spikes above its baseline

## Create alerts from the Recommendations tab

The fastest way to set up alerts is from the **Recommendations** tab in [Error Tracking](https://app.posthog.com/error_tracking). The **Alert coverage** card shows which alert types you've configured and lets you create new alerts directly.

Click **Create alert** next to any unconfigured trigger to open the setup wizard. Choose your destination and configure settings. Since you selected the trigger from the card, the wizard skips the trigger selection step.

## Create alerts from the Configuration page

Alternatively, you can create alerts from [error tracking's configuration page](https://app.posthog.com/error_tracking/configuration#selectedSetting=error-tracking-alerting). Click **Alerting** to see a list of existing alerts, then click **New notification** to create a new one.

## Spike detection

The **Issue spiking** trigger works differently from the other two triggers. Instead of firing when an issue is first seen or reopened, spike alerts trigger when an existing issue's error rate significantly exceeds its historical baseline. This is useful for catching regressions after deploys.

See the [spike detection guide](/docs/error-tracking/spikes) to learn how spike detection works and how to configure sensitivity thresholds.

## Other alerting options

Since error tracking works by capturing `$exception` events, PostHog features that trigger by events can play a role in alerts too.

### Real time destinations

The first way is using [real time destinations](/docs/cdp/destinations). This enables you to send events (like `$exception`) to other tools as soon as they are ingested.

To create a real time destination, go to the [data pipelines tab](https://app.posthog.com/data-management/destinations) in PostHog, click **+ New**, and then select **Destination**. Choose your destination and press **+ Create**.

On the destination creation screen, make sure to add an event matcher for the `$exception` event, filter for the properties you want, and set the trigger options.

### Trend alerts

You can also visualize your `$exception` events using [trends](/docs/product-analytics/trends/overview). Once you create a trend insight, click the **Alerts** button at the top of the insight and then **New alert**.

Here you can set alerts for event volume value, increase, or decrease.
