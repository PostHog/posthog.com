## Download and extract symbol sets

You can download and extract uploaded symbol sets from PostHog to verify what was uploaded, debug source map issues, or compare against your build artifacts.

You can also download symbol sets directly from the [Symbol Sets configuration](https://app.posthog.com/error_tracking/configuration#selectedSetting=error-tracking-symbol-sets) page in the PostHog app by clicking the download button on any uploaded symbol set.

### Download from PostHog

Use `symbol-sets download` to download a symbol set and extract its contents. You can look up the symbol set by ID or by ref:

```bash
# Download by symbol set ID
posthog-cli symbol-sets download --id <symbol-set-id> -o ./output-dir

# Download by symbol set ref
posthog-cli symbol-sets download --ref <symbol-set-ref> -o ./output-dir
```

Find the symbol set ID or ref on the [Symbol Sets configuration](https://app.posthog.com/error_tracking/configuration#selectedSetting=error-tracking-symbol-sets) page.

The command auto-detects the data type and extracts files with the correct extensions:

- **SourceAndMap** – `.js` and `.js.map` files
- **HermesMap** – `.hbc.map` files
- **ProguardMapping** – `.txt` files
- **AppleDsym** – `.dSYM` files

### Extract a local file

Use `symbol-sets extract` to decompress and extract a local symbol set binary file. This doesn't require API authentication:

```bash
posthog-cli symbol-sets extract ./path/to/symbol-set-file -o ./output-dir
```

This is useful for inspecting symbol sets before uploading or comparing local files against what was uploaded to PostHog.
