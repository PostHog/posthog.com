Endpoints require a [Personal API Key](/docs/api#personal-api-keys) for authentication.

## Creating an API key

1. Go to [Personal API Keys](https://us.posthog.com/settings/user-api-keys) in your PostHog settings
2. Click **Create personal API key**
3. Give it a name (e.g., "Endpoints")
4. Select the required scopes:
   - `endpoint:read` - List and execute endpoints
   - `endpoint:write` - Create and update endpoints and their underlying queries
5. Click **Create key** and copy it somewhere safe

## Using the API key

Pass your API key in the `Authorization` header as a Bearer token:

```bash
curl \
  -H "Authorization: Bearer phx_1a2b3c4d5e6f..." \
  "<ph_app_host>/api/projects/{project_id}/endpoints/{endpoint_name}/run"
```

## Security best practices

- **Don't expose keys in client-side code** - API keys should only be used server-side
- **Use minimal scopes** - If you're only executing endpoints, only assign the `endpoint:read` scope to the key.
- **Rotate keys periodically** - Delete old keys and create new ones if you suspect they've been compromised
- **Use separate keys per environment** - Create different keys for development, staging, and production
